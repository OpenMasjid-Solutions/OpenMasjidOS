# APP_MANIFEST_SPEC.md — OpenMasjidAPPS Contract

> This document defines the complete contract between OpenMasjidOS (the platform) and OpenMasjidAPPS (the app catalog). Any app submitted to OpenMasjidAPPS must comply with this specification. OpenMasjidOS enforces these rules at install time and will refuse to install apps that violate them.

---

## 1. OpenMasjidAPPS Repository Structure

The `OpenMasjidAPPS` repository is a flat collection of app definitions, one directory per app, plus a generated catalog index.

```
OpenMasjidAPPS/
├── catalog.json                    # Auto-generated by CI. Never edit by hand.
├── README.md
│
├── prayer-times-display/
│   ├── manifest.yaml               # Required. The app definition.
│   ├── docker-compose.yml          # Required. The app's container definition.
│   ├── icon.svg                    # Required. Square SVG icon, min 256×256 viewport.
│   ├── screenshot-1.png            # Optional. Up to 5 screenshots, 16:9 preferred.
│   ├── screenshot-2.png
│   └── README.md                   # Optional. Developer notes; not shown in UI.
│
├── donation-kiosk/
│   ├── manifest.yaml
│   ├── docker-compose.yml
│   ├── icon.svg
│   └── screenshot-1.png
│
└── ...
```

### Naming rules

- Directory names must match the `id` field in `manifest.yaml` exactly.
- Directory names (and therefore IDs) must be lowercase kebab-case: `[a-z0-9-]+`.
- IDs must be globally unique across the catalog.
- No special characters, no underscores, no spaces.

### CI pipeline

On every push to `main`, a GitHub Actions workflow:
1. Validates every `manifest.yaml` against this spec.
2. Validates every `docker-compose.yml` for required labels.
3. Assembles `catalog.json` from all valid manifests.
4. Fails the build if any app is invalid (the invalid app is excluded with an error, not silently skipped — maintainers must fix it).

## 2. catalog.json Format

`catalog.json` is the single file OpenMasjidOS fetches to populate the App Store. It is regenerated by CI and committed to the repository root, then served via GitHub raw content or release assets.

### Top-level structure

```json
{
  "version": "1",
  "generated_at": "2026-06-18T12:00:00Z",
  "apps": [ /* array of AppEntry objects */ ]
}
```

| Field          | Type   | Description                                           |
|----------------|--------|-------------------------------------------------------|
| `version`      | string | Schema version. Currently `"1"`. Increment on breaking changes. |
| `generated_at` | string | ISO 8601 UTC timestamp of when CI generated this file. |
| `apps`         | array  | Ordered array of AppEntry objects (see below).        |

### AppEntry object

Each element of `apps` is a flattened, enriched version of the app's `manifest.yaml`. The CI pipeline adds asset URLs so OpenMasjidOS never has to construct GitHub raw URLs itself.

```json
{
  "id": "prayer-times-display",
  "name": "Prayer Times Display",
  "tagline": "A beautiful prayer clock for your masjid's screens",
  "description": "Full markdown description...",
  "category": "displays",
  "version": "1.0.0",
  "author": "OpenMasjidAPPS",
  "license": "MIT",
  "icon_url": "https://raw.githubusercontent.com/OpenMasjidOS/OpenMasjidAPPS/main/prayer-times-display/icon.svg",
  "screenshot_urls": [
    "https://raw.githubusercontent.com/OpenMasjidOS/OpenMasjidAPPS/main/prayer-times-display/screenshot-1.png"
  ],
  "manifest_url": "https://raw.githubusercontent.com/OpenMasjidOS/OpenMasjidAPPS/main/prayer-times-display/manifest.yaml",
  "compose_url": "https://raw.githubusercontent.com/OpenMasjidOS/OpenMasjidAPPS/main/prayer-times-display/docker-compose.yml",
  "uses_profile": ["name", "latitude", "longitude", "calc_method", "asr_madhab", "timezone"],
  "settings": [
    {
      "key": "SCREEN_ORIENTATION",
      "label": "Screen orientation",
      "type": "select",
      "options": ["landscape", "portrait"],
      "default": "landscape",
      "required": false,
      "description": "Choose how the display is mounted."
    }
  ],
  "ports": [
    {
      "container": 80,
      "label": "Web interface"
    }
  ],
  "resources": {
    "memory_hint": "128M"
  }
}
```

All fields mirror `manifest.yaml` (see Section 3). The CI pipeline adds `icon_url`, `screenshot_urls`, `manifest_url`, and `compose_url` by constructing GitHub raw URLs from the repo path and commit SHA.

OpenMasjidOS caches `catalog.json` locally and refreshes it on a configurable interval (default: 6 hours) or on explicit user refresh.

## 3. manifest.yaml — Full Specification

### Annotated example

```yaml
# ─── Identity ────────────────────────────────────────────────────────────────

id: prayer-times-display
# Type: string (kebab-case, [a-z0-9-]+)
# Required: yes
# Must match the directory name exactly.
# Once published, IDs are permanent. Do not rename a published app.

name: Prayer Times Display
# Type: string
# Required: yes
# Human-readable display name. Shown in the App Store listing and dashboard.
# Max 64 characters. Plain text only (no markdown, no HTML).

tagline: A beautiful prayer clock for your masjid's screens
# Type: string
# Required: yes
# One short sentence. Shown under the app name in cards.
# Max 120 characters. No period at the end.

description: |
  Show accurate prayer times on any screen in your masjid — a TV in the
  foyer, a monitor in the prayer hall, a tablet at the entrance.

  Automatically uses your masjid's location and calculation method.
  Supports Hanafi and Shafi'i Asr times. Displays Hijri date alongside
  Gregorian. Supports landscape and portrait orientations.

  No internet connection required after the first setup.
# Type: string (markdown)
# Required: yes
# Shown in full on the app detail page. Markdown is rendered.
# Keep it focused on what the app does and why it's useful.
# Do NOT include installation instructions here.

category: displays
# Type: string (enum)
# Required: yes
# See Section 6 for the full category list and descriptions.

version: 1.0.0
# Type: string (semver: MAJOR.MINOR.PATCH)
# Required: yes
# Follows semantic versioning. Bump PATCH for bug fixes, MINOR for new
# features, MAJOR for breaking changes to how the app is configured.

author: OpenMasjidAPPS
# Type: string
# Required: yes
# The author or organization name. Shown on the detail page.

license: MIT
# Type: string
# Required: yes
# SPDX license identifier. E.g. MIT, Apache-2.0, GPL-3.0-only.

# ─── Assets ──────────────────────────────────────────────────────────────────

icon: icon.svg
# Type: string (filename)
# Required: yes
# Must be a file present in the same directory as manifest.yaml.
# Accepts: icon.svg (preferred), icon.png
# SVG: square viewport, min effective 256×256. Fills a rounded square container.
# PNG: minimum 256×256 pixels, ideally 512×512, transparent background.

screenshots:
  - screenshot-1.png
  - screenshot-2.png
# Type: list of strings (filenames)
# Required: no (recommended)
# Files must be present in the same directory.
# Maximum 5 screenshots. 16:9 aspect ratio strongly preferred.
# PNG or JPEG. Minimum 1280×720 pixels.

# ─── Profile integration ─────────────────────────────────────────────────────

uses_profile:
  - name
  - latitude
  - longitude
  - calc_method
  - asr_madhab
  - timezone
# Type: list of strings (enum values)
# Required: no
# Declares which masjid profile fields this app needs.
# OpenMasjidOS injects these as environment variables when the app starts.
# See Section 4 for the full list of injectable profile fields and their
# corresponding environment variable names.
# Only list fields you actually use — the UI shows the user what you access.

# ─── User-configurable settings ──────────────────────────────────────────────

settings:
  - key: SCREEN_ORIENTATION
    label: Screen orientation
    description: Choose how the display screen is physically mounted.
    type: select
    options:
      - landscape
      - portrait
    default: landscape
    required: false

  - key: PRIMARY_COLOR
    label: Accent colour
    description: The highlight colour used for prayer names and times.
    type: text
    default: "#1FA37A"
    required: false

  - key: DISPLAY_SECONDS
    label: Show seconds
    description: Show a live seconds counter alongside the current time.
    type: boolean
    default: false
    required: false

  - key: FONT_SIZE
    label: Font size (px)
    description: Base font size for prayer times. Larger screens may need a bigger value.
    type: number
    min: 12
    max: 120
    default: 48
    required: false

  - key: ADMIN_PASSWORD
    label: Display admin password
    description: Password to access the display's own settings screen.
    type: password
    required: true
# Type: list of Setting objects
# Required: no
# Shown to the user in an "Install settings" form before install.
# Values are injected as environment variables (key = env var name).
# See Section 5 for the full Setting type reference.

# ─── Ports ───────────────────────────────────────────────────────────────────

ports:
  - container: 80
    label: Web interface
  - container: 9090
    label: Admin panel
# Type: list of Port objects
# Required: no (but strongly recommended if the app has a UI)
# Informs OpenMasjidOS which ports to expose and what to label them
# in the "Open" button on the dashboard. The platform automatically
# assigns a free host port and maps it to container port.
# See Section 7.

# ─── Resource hints ──────────────────────────────────────────────────────────

resources:
  memory_hint: 128M
  cpu_hint: 0.25
# Type: object
# Required: no
# Soft hints shown to the user during install ("This app uses about 128 MB of RAM").
# Not enforced as hard Docker limits unless the user enables resource limits
# in the platform settings. See Section 8.
```

## 4. Environment Variable Injection (Profile Values)

When OpenMasjidOS installs or starts an app, it reads the masjid profile stored in the platform's config and injects the requested fields as environment variables into the app's containers.

The app declares which profile fields it needs via `uses_profile`. OpenMasjidOS injects only the declared fields — no extras.

### Profile fields and their environment variable names

| Profile field   | Environment variable      | Type    | Example value                  | Description |
|-----------------|---------------------------|---------|-------------------------------|-------------|
| `name`          | `MASJID_NAME`             | string  | `"Al-Noor Islamic Centre"`    | Full masjid name. |
| `address`       | `MASJID_ADDRESS`          | string  | `"123 High Street, London"`   | Street address. |
| `latitude`      | `MASJID_LATITUDE`         | float   | `"51.5074"`                   | Decimal degrees, negative = south. |
| `longitude`     | `MASJID_LONGITUDE`        | float   | `"-0.1278"`                   | Decimal degrees, negative = west. |
| `calc_method`   | `MASJID_CALC_METHOD`      | string  | `"MWL"`                       | Prayer time calculation method. See below. |
| `asr_madhab`    | `MASJID_ASR_MADHAB`       | string  | `"hanafi"` or `"shafii"`      | Asr shadow ratio. |
| `timezone`      | `MASJID_TIMEZONE`         | string  | `"Europe/London"`             | IANA timezone string. |
| `language`      | `MASJID_LANGUAGE`         | string  | `"en"` or `"ar"`              | BCP 47 language tag. |
| `country`       | `MASJID_COUNTRY`          | string  | `"GB"`                        | ISO 3166-1 alpha-2 country code. |
| `currency`      | `MASJID_CURRENCY`         | string  | `"GBP"`                       | ISO 4217 currency code. Used by donation apps. |
| `phone`         | `MASJID_PHONE`            | string  | `"+44 20 7946 0958"`          | Contact phone. |
| `email`         | `MASJID_EMAIL`            | string  | `"info@alnoor.org.uk"`        | Contact email. |
| `website`       | `MASJID_WEBSITE`          | string  | `"https://alnoor.org.uk"`     | Website URL. |

### Calculation method values (`calc_method`)

| Value    | Organisation |
|----------|-------------|
| `MWL`    | Muslim World League |
| `ISNA`   | Islamic Society of North America |
| `Egypt`  | Egyptian General Authority of Survey |
| `Makkah` | Umm Al-Qura University, Makkah |
| `Karachi`| University of Islamic Sciences, Karachi |
| `Tehran` | Institute of Geophysics, University of Tehran |
| `Jafari` | Shia Ithna Ashari (Jafari) |
| `UOIF`   | Union des organisations islamiques de France |
| `ISWA`   | Islamic Society of Washington Area |
| `MF`     | Moonsighting.com (Khalid Shaukat) |

### User-configured settings injection

User settings (from the `settings` array) are injected as environment variables using the `key` field directly. For example, if the user sets `SCREEN_ORIENTATION=portrait`, the container receives `SCREEN_ORIENTATION=portrait`.

### Injection mechanism

OpenMasjidOS writes all injected values into an `.env` file at:

```
/opt/openmasjid/apps/<app-id>/.env
```

Docker Compose automatically picks up this file. Apps must not hardcode the path — they receive the values as normal environment variables.

The `.env` file is regenerated every time the app is started or restarted. If the masjid profile is updated, the admin is prompted to restart affected apps to pick up the new values.

## 5. Settings Types Reference

Settings appear in the "Configure before installing" form in the UI. Each setting maps to one environment variable in the running container.

### Common fields (all setting types)

| Field         | Type    | Required | Description |
|---------------|---------|----------|-------------|
| `key`         | string  | yes      | The environment variable name. UPPER_SNAKE_CASE. `[A-Z0-9_]+`. |
| `label`       | string  | yes      | Human-readable label shown in the form. Max 64 chars. |
| `description` | string  | no       | Short help text shown under the field. Max 200 chars. |
| `type`        | string  | yes      | One of: `text`, `select`, `number`, `boolean`, `password`. |
| `default`     | any     | no       | Default value. Must match the type. |
| `required`    | boolean | no       | If `true`, the user must provide a value before installing. Default: `false`. |

### `type: text`

A single-line text input.

```yaml
- key: DISPLAY_TITLE
  label: Display title
  description: Shown at the top of the prayer times screen.
  type: text
  default: "Prayer Times"
  required: false
  # Optional extra constraints:
  min_length: 1
  max_length: 80
  pattern: "^[^<>]*$"   # regex; validated before install
```

| Extra field   | Type   | Description |
|---------------|--------|-------------|
| `min_length`  | int    | Minimum character count. |
| `max_length`  | int    | Maximum character count. |
| `pattern`     | string | ECMAScript regex the value must match. |

### `type: select`

A dropdown with a fixed list of choices.

```yaml
- key: THEME
  label: Colour theme
  type: select
  options:
    - light
    - dark
    - auto
  default: dark
  required: true
```

| Extra field | Type          | Description |
|-------------|---------------|-------------|
| `options`   | list(string)  | Required. The allowed values. Must have at least 2 entries. |

The UI renders each option as its raw value unless `option_labels` is also provided:

```yaml
  option_labels:
    light: "Light (bright background)"
    dark: "Dark (night mode)"
    auto: "Follow system setting"
```

### `type: number`

A numeric input. Stored and injected as a string (all env vars are strings).

```yaml
- key: FONT_SIZE
  label: Font size (px)
  type: number
  default: 48
  min: 12
  max: 120
  step: 1
  required: false
```

| Extra field | Type  | Description |
|-------------|-------|-------------|
| `min`       | float | Minimum value (inclusive). |
| `max`       | float | Maximum value (inclusive). |
| `step`      | float | Step increment for the UI spinner. Default: `1`. |

### `type: boolean`

A toggle switch. Injected as `"true"` or `"false"` (strings).

```yaml
- key: SHOW_HIJRI_DATE
  label: Show Hijri date
  description: Display the Islamic calendar date alongside the Gregorian date.
  type: boolean
  default: true
  required: false
```

### `type: password`

A password input. Masked in the UI. The value is stored in the `.env` file on disk — it is not encrypted at rest beyond normal filesystem permissions. Do not use for secrets that require higher protection.

```yaml
- key: ADMIN_PASSWORD
  label: Admin password
  description: Set a password to protect the display settings page.
  type: password
  required: true
  min_length: 8
```

Supports `min_length` and `max_length` like `type: text`. Pattern matching is supported but not recommended for passwords (avoid leaking constraints that weaken security).

## 6. Category Definitions

Every app must belong to exactly one category. Categories are displayed as tabs/filters in the App Store.

| Category      | ID            | Description | Example apps |
|---------------|---------------|-------------|--------------|
| Displays      | `displays`    | Apps designed to run on screens in the masjid — prayer times boards, Quran verses, announcements, countdowns. Typically accessed from the display device, not managed day-to-day. | Prayer Times Display, Announcement Board, Adhan Countdown |
| Donations      | `donations`   | Apps that facilitate charitable giving — kiosks, online donation pages, JustGiving integrations, sadaqah jars. | Donation Kiosk, Online Giving Page |
| Community     | `community`   | Apps for member communication, event management, volunteering, and engagement. | Event Calendar, Newsletter, WhatsApp Broadcast Tool |
| Quran         | `quran`       | Apps centred on Quran recitation, study, memorisation, or display. | Quran Recitation Stream, Hifz Tracker |
| Admin         | `admin`       | Tools for masjid staff — attendance tracking, facility booking, committee documents. | Room Booking, Attendance Register |
| Utilities     | `utilities`   | Everything else — infrastructure helpers, integrations, developer tools. | SMTP Relay, Backup Viewer |

When in doubt, choose the category that best describes the app's *primary* audience and use case. An app that shows prayer times on a kiosk and also takes donations is `displays` if the display is its main function, `donations` if giving is its main function.

## 7. Port Declarations

Apps declare which container ports they expose using the `ports` field. OpenMasjidOS uses this information to:

1. Automatically assign a free host port and create the Docker port mapping.
2. Show an "Open" button on the dashboard card with the correct URL.
3. Warn the user if a port is already in use before installing.

```yaml
ports:
  - container: 80
    label: Web interface
    protocol: tcp       # tcp (default) or udp
    default_host: 8080  # optional — preferred host port; platform picks another if taken
```

| Field          | Type   | Required | Description |
|----------------|--------|----------|-------------|
| `container`    | int    | yes      | The port the container listens on. 1–65535. |
| `label`        | string | yes      | Shown in the UI as the button label and in the port mapping list. |
| `protocol`     | string | no       | `tcp` or `udp`. Default: `tcp`. |
| `default_host` | int    | no       | The platform tries this host port first; falls back to a random free port in the range 10000–19999 if unavailable. |

The actual assigned host port is stored in the app's state file and injected into the compose as `OMOS_HOST_PORT_<CONTAINER_PORT>` (e.g. `OMOS_HOST_PORT_80=10042`). Compose files should reference this variable for their port mapping (see Section 9 — docker-compose.yml conventions).

Apps must not hardcode host ports in their `docker-compose.yml`. The platform owns port assignment.

## 8. Resource Hints

Resource hints are informational — they help the admin understand what an app needs before installing, especially on low-power hardware like a Raspberry Pi.

```yaml
resources:
  memory_hint: 128M    # Typical RSS memory usage when running
  cpu_hint: 0.25       # Typical CPU share (0.25 = 25% of one core)
  storage_hint: 500M   # Typical persistent data growth over time
  arch:                # List of supported architectures; omit to mean "all"
    - amd64
    - arm64
```

| Field           | Type         | Required | Description |
|-----------------|--------------|----------|-------------|
| `memory_hint`   | string       | no       | Docker memory size string (`128M`, `1G`). |
| `cpu_hint`      | float        | no       | Fraction of one CPU. `0.5` = half a core. |
| `storage_hint`  | string       | no       | Expected data volume growth over time. |
| `arch`          | list(string) | no       | Supported CPU architectures. Valid values: `amd64`, `arm64`, `arm/v7`. Omit to indicate all. |

If the admin's server has less free memory than `memory_hint`, the platform shows a warning (not a hard block) during install: "This app may need more memory than is currently available. It might run slowly."

If the host architecture is not in `arch`, the platform blocks install and shows a clear message: "This app isn't available for your device's processor type yet."

## 9. docker-compose.yml Conventions

Every app includes a `docker-compose.yml` that OpenMasjidOS renders and runs via `docker compose`.

### Required labels

Every service in the compose file **must** include these labels:

```yaml
services:
  app:
    image: openmasjidapps/prayer-times-display:latest
    labels:
      com.openmasjid.app: prayer-times-display        # must match manifest id
      com.openmasjid.service: app                     # logical service name (kebab-case)
      com.openmasjid.managed: "true"                  # marks this as platform-managed
```

Without `com.openmasjid.app`, OpenMasjidOS cannot find or manage the container. The CI validation step will reject any compose file missing this label.

### Project naming

OpenMasjidOS always starts apps with:

```
docker compose -p omos-<app-id> up -d
```

The compose project name is always `omos-<app-id>`. Apps must not set `name:` at the top of their compose file — the platform overrides it.

### Volume paths

All named volumes must map to paths under `/opt/openmasjid/apps/<app-id>/`:

```yaml
services:
  app:
    volumes:
      - data:/data

volumes:
  data:
    driver: local
    driver_opts:
      type: none
      o: bind
      device: /opt/openmasjid/apps/prayer-times-display/data
```

The platform creates the bind-mount directory before running `docker compose up`. Apps must not create directories outside `/opt/openmasjid/apps/<own-id>/`.

### Environment variable usage

Use the injected variables directly:

```yaml
services:
  app:
    image: openmasjidapps/prayer-times-display:latest
    env_file:
      - /opt/openmasjid/apps/prayer-times-display/.env
    environment:
      # You may also set non-secret static defaults here:
      TZ: ${MASJID_TIMEZONE:-UTC}
```

The platform writes the `.env` file before every `docker compose up`. Using `env_file` is the recommended pattern. Individual `environment:` overrides are allowed for static defaults.

### Port mapping

Use the `OMOS_HOST_PORT_<CONTAINER_PORT>` variable assigned by the platform:

```yaml
services:
  app:
    ports:
      - "${OMOS_HOST_PORT_80:-10080}:80"
```

The fallback value (`:-10080`) is used only during local development when the platform is not managing the compose file. It must not conflict with another app's declared port. In production, the platform always sets the variable.

### Restart policy

All services must use `restart: unless-stopped`:

```yaml
services:
  app:
    restart: unless-stopped
```

Do not use `restart: always` (it restarts even after a deliberate `docker stop`) or `restart: on-failure` (insufficient for a managed service).

### Privileged access and socket mounting

Apps must not request:
- `privileged: true`
- `/var/run/docker.sock` as a volume
- `cap_add` of any capability other than those in the allowed set below
- `network_mode: host`
- `pid: host`

**Allowed `cap_add` values:** `NET_BIND_SERVICE` (to bind ports below 1024). All others require a platform team review and explicit user confirmation during install.

If a manifest requests `/var/run/docker.sock`, OpenMasjidOS will show the user a prominent security warning and require explicit confirmation. The capability must be declared in `manifest.yaml`:

```yaml
# In manifest.yaml — only if truly necessary:
requires_docker_socket: true
```

Apps that request the Docker socket will be marked with a security badge in the App Store and will not be installable without manual confirmation.

## 10. Validation Rules

OpenMasjidOS validates every app before installation. The following checks are enforced. Failure produces a clear, friendly error message and blocks install.

### Manifest validation

| Rule | Error shown to user |
|------|---------------------|
| `id` matches directory name | "This app has a configuration error. Please report it to the app author." |
| `id` matches `[a-z0-9-]+` | (same) |
| `name` present, max 64 chars | (same) |
| `tagline` present, max 120 chars | (same) |
| `description` present | (same) |
| `category` is a known value | (same) |
| `version` is valid semver | (same) |
| `author` present | (same) |
| `license` is a valid SPDX identifier | (same — non-blocking warning if unknown SPDX) |
| `icon` file exists in app directory | "This app is missing its icon. Please report it to the app author." |
| All `screenshots` files exist | Non-blocking warning; screenshots simply not shown. |
| All `uses_profile` values are known profile fields | "This app requests a profile field that doesn't exist. Please report it." |
| All `settings[].key` values are UPPER_SNAKE_CASE | "This app has a configuration error. Please report it." |
| No two settings share the same `key` | (same) |
| Settings `type` is a known type | (same) |
| `select` settings have at least 2 options | (same) |
| `number` settings: `min <= default <= max` (if all provided) | (same) |
| Required settings have no `default` OR user has provided a value | "Please fill in all required settings before installing." |
| `ports[].container` is 1–65535 | "This app has an invalid port setting. Please report it." |
| `resources.arch` values are known | Non-blocking warning. |
| Host architecture is in `resources.arch` (if declared) | "This app doesn't support your device's processor type." |
| `requires_docker_socket` only if explicitly declared | "This app requests access to Docker. You need to confirm this before installing." |

### Compose validation

| Rule | Error shown to user |
|------|---------------------|
| File is valid YAML | "This app has a configuration error (bad compose file). Please report it." |
| All services have `com.openmasjid.app` label matching the manifest `id` | (same) |
| `com.openmasjid.managed: "true"` present on all services | (same) |
| No service uses `privileged: true` | "This app is asking for unrestricted system access, which isn't allowed." |
| No service mounts `/var/run/docker.sock` (unless `requires_docker_socket: true`) | "This app is trying to access Docker directly. This requires your approval." |
| No `cap_add` values outside the allowed set (unless reviewed) | "This app is asking for elevated system permissions, which aren't allowed." |
| No `network_mode: host` or `pid: host` | "This app is asking for host-level network access, which isn't allowed." |
| Volume paths are under `/opt/openmasjid/apps/<id>/` | "This app is trying to store files outside its own folder, which isn't allowed." |
| All services have `restart: unless-stopped` | Non-blocking; platform overrides to `unless-stopped` with a warning in logs. |

### Pre-flight checks (at install time, not catalog time)

| Check | User message |
|-------|-------------|
| Sufficient free disk space (storage_hint × 2, min 100 MB) | "Your device is running low on storage. Free up some space before installing." |
| App ID not already installed | "This app is already installed. You can manage it from your dashboard." |
| No port conflict on declared ports | "Port [X] is already in use by another app. You'll need to stop that app first." |
| Docker daemon is reachable | "Something went wrong connecting to Docker. Try restarting OpenMasjidOS." |
| Image can be pulled (network reachable) | "Could not download this app. Check your internet connection and try again." |

## 12. Platform-injected Variables (always available)

Regardless of `uses_profile`, OpenMasjidOS always injects the following into every app's environment:

| Variable                | Description |
|-------------------------|-------------|
| `OMOS_APP_ID`           | The app's `id` from its manifest. |
| `OMOS_APP_VERSION`      | The app's `version` from its manifest. |
| `OMOS_PLATFORM_VERSION` | The OpenMasjidOS core version running. |
| `OMOS_DATA_DIR`         | Absolute path to the app's data directory: `/opt/openmasjid/apps/<id>`. |
| `OMOS_HOST_PORT_<N>`    | The host port mapped to container port `N`, for each declared port. |

Apps may use these for telemetry, version display, or self-configuration. They must not be used to escape the app's sandbox.

## 13. Versioning and Updates

When an app is updated in `OpenMasjidAPPS`:

1. The app author bumps `version` in `manifest.yaml` following semver.
2. CI regenerates `catalog.json` with the new version.
3. OpenMasjidOS detects the version change on its next catalog refresh and marks the app as "Update available" in the dashboard.
4. The admin taps "Update". OpenMasjidOS runs `docker compose pull` followed by `docker compose up -d --force-recreate`.
5. The existing data volume is preserved. The `.env` file is regenerated (picking up any new settings or profile changes).
6. If the update fails (image pull failure, compose error), OpenMasjidOS automatically rolls back to the previous image tag and shows a friendly error.

**Backward compatibility:** A MINOR or PATCH version bump must not remove or rename existing `settings` keys or `uses_profile` fields without a MAJOR bump. Removing a setting that a user has configured will cause the app to lose that configuration silently — treat it as a breaking change.
